When should you use a Python DI container?

Injex is for shared service graphs across APIs, CLIs, workers, and tests. It is not meant to replace every Python dependency management style.

NeedGood fit
A few dependencies in one entrypointManual wiring
Request dependencies inside one frameworkFramework dependency system
Shared service graph across API, CLI, worker, and testsInjex
Typed constructor injection without provider DSLInjex
Complex provider/configuration frameworkLarger DI container

Choose Injex when

  • You want a small API surface.
  • Constructors already express dependencies with type hints.
  • Tests need temporary overrides.
  • Startup should validate the graph before real objects are built.

Skip Injex when

  • Manual wiring is still clear.
  • Every entrypoint is already managed by a framework.
  • You need a rich provider/configuration DSL.
