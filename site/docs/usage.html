Usage scenarios

One service graph across multiple Python entrypoints.

Service layer wiring

Wire repositories, gateways, clients, and use cases once at startup. Application code can depend on normal Python types while startup owns composition.

CLI applications

CLI commands often share settings, API clients, repositories, and service objects. A container keeps setup in one place without module-level singletons.

Background workers

Workers often need long-lived clients and short-lived per-job state. Register clients as singletons and create one scope per job or message.

Tests

Use override() to replace slow or external services with fakes inside one specific test block.

Startup validation

Use validate() or assert_valid() in startup checks and CI smoke tests.
