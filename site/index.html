Zero dependencies · Typed wiring · Framework-agnostic

Injex

Tiny typed dependency injection for Python apps that want explicit wiring without a framework-sized container.

GitHub PyPI Docs

Small API

Register services, factories, instances, and lifetimes without a provider DSL or runtime dependencies.

Typed injection

Constructors use normal Python type hints, so application code stays explicit and framework-independent.

Testable by design

Use temporary overrides and graph validation to catch wiring errors before startup or slow integration tests.

Quick start

from injex import Container

class Repository:
    pass

class Service:
    def __init__(self, repo: Repository):
        self.repo = repo

container = Container()
container.add_singleton(Repository)
container.add_transient(Service)
container.assert_valid()

service = container.resolve(Service)
